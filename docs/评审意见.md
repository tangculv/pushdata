## 6. 致开发团队 (To Dev Team) - ⚡️ 避坑指南

作为产品总监，针对即将开发的版本，我再强调几个**必须执行**的技术细节，请开发同学务必 verify：

1.  **Excel 日期陷阱**：
    *   一定要找带 `cellDates: true` 或者类似功能的库配置，或者手动实现 `(value - 25569) * 86400 * 1000` 的转换算法。
    *   测试用例必须覆盖：`2026-01-23` (文本), `2026/01/23` (文本), `45231.5` (数字) 三种情况。

2.  **业务主键提取**：
    *   PRD 第 5 节的表头行号 (0-based) 务必做成**配置项**（Configurable），不要写死在代码里。SaaS 平台导出的 Excel 偶尔会加一行“温馨提示”在顶部，导致 Row Index 偏移，要留后路。

3.  **熔断恢复**：
    *   触发熔断（连续失败 5 次）后，UI 必须给用户一个“重试”按钮。不要让用户只能重启软件来恢复。

4.  **文件占用**：
    *   读取 Excel 时，注意文件可能被用户用 Excel 打开了（Windows 下文件锁）。此时读取会报错，请 Catch 这个错误并友好提示“请关闭 Excel 后再试”。
